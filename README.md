**Проект 1. Регрессионное тестирование**

Выполнение регрессионного тестирования сервиса Mesto по готовым тест-кейсам. Задачи: знакомство с функциональностью сервиса, тестирование по тест-кейсам с проставлением статусов выполнения проверок, оформление баг-репортов по выявленным дефектам. [https://docs.google.com/spreadsheets/d/1Gf8UqvSybFTYJZLHdfQxNv-qYjS9QiKB2NxxhYYsfJE/edit?usp=drive_link](https://docs.google.com/spreadsheets/d/1Gf8UqvSybFTYJZLHdfQxNv-qYjS9QiKB2NxxhYYsfJE/edit?usp=drive_link)

**Проект 2. Яндекс Маршруты**

Регрессионное тестирование части сервиса построения маршрутов (валидация полей, логика расчета стоимости и времени поездки на собственном автомобиле). Составлены тест-кейсы с применением техник тест-дизайна, оформлены баг-репорты по выявленным дефектам. По итогам тестирования подготовлен тестовый набор. [https://docs.google.com/spreadsheets/d/1G67F3FTKN-PX68XDYOK1ZbJux_fe-Xll5esODcecOpk/edit?usp=sharing](https://docs.google.com/spreadsheets/d/1G67F3FTKN-PX68XDYOK1ZbJux_fe-Xll5esODcecOpk/edit?usp=sharing)

**Проект 3. Яндекс Маршруты +каршеринг**

Выполнение проекта по тестированию интерфейса и функциональности сервиса бронирования автомобилей. Изучены требования и макеты интерфейса. Подготовлен чек-лист для проверки вёрстки формы бронирования и элементов навигационной карты (иконок автомобилей и действий с ними). Учтены различные состояния интерфейсных элементов, включая поле «Добавить права», а также окна «Машина забронирована», «Вы уверены, что хотите отменить поездку?» и «Поездка отменена».  
Подготовлена тестовая документация: чек-листы для окон «Способ оплаты» и «Добавление карты», тест-кейсы для кнопки «Забронировать». Проведено тестирование приложения на тестовом стенде. Выполнено кроссбраузерное тестирование с использованием DevTools. [https://docs.google.com/spreadsheets/d/1puNeQlKigIDkI-wGZb9mle0Ax8qVWU0m2BX_XV78nS4/edit?usp=sharing](https://docs.google.com/spreadsheets/d/1puNeQlKigIDkI-wGZb9mle0Ax8qVWU0m2BX_XV78nS4/edit?usp=sharing)

**Проект 4. Мобильное приложение «Яндекс Метро»**

Выполнение проекта по тестированию мобильного приложения «Яндекс Метро». Подготовлен чек-лист для функционального тестирования требований, затронутых рефакторингом. Для регрессионного тестирования составлен чек-лист с учётом особенностей мобильного приложения: проверок работы сети, геолокации, переключения между приложениями, входящих звонков и уведомлений, смены ориентации экрана, режима энергосбережения и уровня заряда аккумулятора. Проведено тестирование приложения на эмуляторе Android Studio  на устройстве  Honor 8 (Android 9.0, разрешение 1080×1920). Выявленные дефекты оформлены в виде баг-репортов в Google Таблице. По результатам тестирования подготовлен отчёт. [https://docs.google.com/spreadsheets/d/19mhow9gINmjUDb3tmNs_JAvFIJ1fLfsYcuzrojntOVM/edit?usp=sharing](https://docs.google.com/spreadsheets/d/19mhow9gINmjUDb3tmNs_JAvFIJ1fLfsYcuzrojntOVM/edit?usp=sharing)

**Проект 5. API "Яндекс Прилавка"**

Тестирование новой функциональности API веб-приложения доставки продуктов. Выполнены тестовые запросы к API с использованием инструмента Postman. Оформлены баг-репорты по выявленным дефектам и подготовлен отчет о результатах тестирования. Тестирование веб- и мобильного приложения доставки самокатов.
